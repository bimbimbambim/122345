# 🟠 UX-ПРОБЛЕМЫ, ВЛИЯЮЩИЕ НА КОНВЕРСИЮ / УДЕРЖАНИЕ

1. **Кнопки «Все» — пустые** (#8): пользователь исследует контент, жмёт «Все» — ничего. Снижает доверие и engagement.
2. **Нет ошибки при неудачном платеже** (#35): молчаливый провал = пользователь думает деньги списались, приложение зависло.
3. **`capture="user"`** (#39): нельзя выбрать фото из галереи. Исключает сценарий «у меня есть хорошее фото».
4. **Tier без описания** (#40): пользователь не понимает за что платит — конверсия в premium tier падает.
5. **Бесконечный polling без таймаута** (#46): зависшая генерация = пользователь закрывает приложение.
6. **Fake прогресс 92%** (#44): потеря доверия если ждать 60+ секунд на 92%.
7. **UploadModal visual quality** (#37): первое взаимодействие с продуктом — худший по виду экран.

# ♿ ACCESSIBILITY-ПРОБЛЕМЫ

1. **Нет `aria-label` ни на одной кнопке** — иконочные кнопки (X, Download, Share, Copy, +) не имеют текстового описания для screen reader.
2. **`user-select: none` на всём приложении** (index.css, line 13) — запрещает копирование текста, включая реферальные ссылки (хотя есть кнопка Copy).
3. **Цвет `text-white/25`, `text-white/20`, `text-white/30`** — opacity 20-30% на тёмном фоне даёт контраст ~1.5:1, что не проходит WCAG AA (минимум 4.5:1 для мелкого текста).
4. **Текст `text-[9px]` и `text-[10px]`** — размеры ниже минимально рекомендуемых 11px для mobile.
5. **Focus-состояния не определены** — нет `:focus-visible` стилей ни на одном интерактивном элементе.
6. **`-webkit-touch-callout: none`** отключает контекстное меню — пользователи не могут сохранить/скопировать ссылку длинным нажатием.
7. **Анимации без `prefers-reduced-motion`** — пульсации, float, shimmer крутятся постоянно, игнорируя системные настройки.

# 📱 АДАПТИВНЫЕ ПРОБЛЕМЫ

1. **4-колоночный Quick Actions grid** (#11): на экране 320px (iPhone SE 1st gen) 4 колонки слишком тесные.
2. **StyleCard `size="md"` w-[164px]** — фиксированная ширина в горизонтальном скролле не адаптируется к zoom/text size настройкам.
3. **Hero banner `minHeight: 168`** — фиксированная высота на маленьких экранах обрезает контент.
4. **BottomNav `-mt-5`** на центральной кнопке — на Android Chrome с изменённым размером шрифта кнопка может выходить за пределы.
5. **`pb-28` (padding-bottom: 7rem)** на всех страницах — фиксированный отступ не учитывает реальную высоту BottomNav с safe area.
6. **PaywallModal `maxHeight: 88vh`** — на телефонах с большой системной клавиатурой (если открыта) остаётся мало места.
7. **Desktop (браузер)** — приложение полностью нефункционально в браузере без Telegram, показывает demo-state но без адаптации к desktop layout.
