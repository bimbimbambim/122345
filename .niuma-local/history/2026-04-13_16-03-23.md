## ✅ Админ-панель готова

### Что сделано

**Backend (`backend/`):**
- **`Style` модель** в БД (`models.py`) — хранит все стили: название, категория, картинка, цены, теги, промпт, флаги (trending/popular/new/active)
- **`seed_styles.py`** — 10 стилей из текущего frontend-кода автоматически загружаются в БД при первом запуске
- **`GET /api/styles`** — публичный endpoint для frontend (стили из БД вместо статического файла)
- **`POST /api/admin/login`** — авторизация двумя способами:
  - `admin_token` — длинный секрет из `.env` (`ADMIN_TOKEN`)
  - `init_data` — Telegram WebApp initData (проверяется через HMAC)
- **`ADMIN_TELEGRAM_IDS`** — comma-separated список TG ID в `.env`, только они получают доступ
- **CRUD `/api/admin/styles`** — GET/POST/PUT/DELETE стилей
- **`/api/admin/users`** — список с поиском + действия: ban/unban, начислить монеты, выдать/снять права администратора
- **`/api/admin/generations`** — лог генераций с фильтрацией по статусу

**Admin SPA (`admin/` — порт 5175):**
- 🔐 **Login** — вход по `ADMIN_TOKEN` из `.env`
- 📊 **Dashboard** — 6 ключевых метрик (пользователи, генерации, выручка, подписки, стили)
- 🎨 **Styles** — таблица всех стилей с превью, редактирование/удаление/скрытие через модальные формы
- 👥 **Users** — список с поиском, управление: бан, монеты, права
- 🖼️ **Generations** — лог с фильтром по статусу и превью результатов

### Настройка `.env`
```env
ADMIN_TELEGRAM_IDS=123456789   # твой Telegram ID
ADMIN_TOKEN=super-secret-token  # токен для входа в панель
```

```bash
cd admin && npm run dev   # http://localhost:5175

```

**Панель открыта в браузере** → введи `ADMIN_TOKEN` из `.env` для входа.
